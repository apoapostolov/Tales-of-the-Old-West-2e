# Book 04 Chapter Rewrite Template

Use this template for every player-facing chapter in `04-trials-of-the-old-west`.

## Chapter Shape

1. **Opening voice.** Start with a concrete historical pressure, not a design summary. A reader should know what the chapter feels like before learning what it tracks.
2. **When to use it.** State the table situation that calls for the chapter. Do not call it a framework, subsystem, primitive, engine, or dial.
3. **What the table tracks.** Name the few gauges, resource dice, scores, or clocks the chapter needs. Explain each in plain book language.
4. **The procedure.** Give the GM a runnable loop:
   - trigger;
   - player choices;
   - GM setup;
   - roll or non-roll resolution;
   - success ladder;
   - failure cost;
   - downstream state change;
   - what decision the result creates next.
5. **Examples and consequences.** Show one short worked example that uses the actual rules. Tie the result to Standing, Wanted, Refuge, Capital, Trouble, Faith, business, faction, season, or town consequences where relevant.
6. **GM tools.** Include only tables that are roll lookups or compact references. Convert choice menus and actions into bolded paragraph entries.
7. **Closing weight.** End on a concrete consequence or a hook for play, not a design recap.

## Publication Voice Rules

- Use Book 02 as the voice model: material fact, period texture, clear procedure.
- Write to GMs and players, not designers.
- Use chapter references as book text: "the faction chapter," "Chapter 13," "the outlaw rules." Do not mention paths, skill files, or internal design notes.
- Replace internal vocabulary:
  - engine -> rules, procedure, game, chapter, table
  - framework -> rules, chapter, procedure
  - dial -> gauge, choice, rating, score, track, die
  - primitive -> rule, pattern, procedure
  - propagation -> the faction's allies and enemies notice
  - decay valve -> fades, cools, drops with neglect
  - metaplot -> decade, national weather, campaign year
- Use ALL-CAPS for abilities and game terms where the corebook does.
- Keep mechanics compatible with existing gauges and the core operation rules.

## Rules Acceptance Checklist

- [ ] The GM knows when to use the chapter.
- [ ] The players have at least one meaningful choice before the main roll.
- [ ] The roll or procedure says who rolls, what ability applies, what modifiers matter, and whether it can be pushed.
- [ ] The result ladder distinguishes failure, limited success, full success, and exceptional success where appropriate.
- [ ] Failure produces playable consequences, not a dead end.
- [ ] Success produces future opportunities, obligations, or risks, not only money.
- [ ] The chapter does not create free Capital, free Standing, free Influence, or unbounded dice bonuses.
- [ ] No always-best option exists in a menu.
- [ ] The GM is not asked to solve the system by fiat after the roll.
- [ ] The example of play actually uses the procedure as written.

## Balance Checklist

- [ ] What behavior does this reward?
- [ ] What will players believe it rewards?
- [ ] What will the GM need to manage?
- [ ] Is there a dominant strategy?
- [ ] Is there a false choice?
- [ ] Can the rule be farmed over seasons?
- [ ] Does it increase bookkeeping without increasing decisions?
- [ ] Does it break the expected 3-to-6-check operation tempo?
- [ ] Does it stack with existing bonuses beyond the normal `+3` modifier discipline?
- [ ] Does it preserve the Old West game's scarcity, distance, and consequence?
