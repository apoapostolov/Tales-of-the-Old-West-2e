# Naming the Game — Flavorful Generic and Genre-Specific Language

> **STATUS: FILLED.** This is the naming reference for creating new YZE games from the engine. Load it whenever a design is moving from analysis into a new game's actual vocabulary: attributes, skills, races, life events, Willpower, harm names, talents, holdings, powers, gear, factions, travel jobs, and special rules. Its job is to prevent sterile framework words from leaking into rules text.

## 1. Why this file exists

`yze-design` needs analytical language when it studies the engine. Designers need terms like pressure, cost, pattern, layer, and balance when they are thinking. A game book needs living names. Players do not want a "protected dial," a "metacurrency," a "domain control system," or a "resource extraction loop." They want **Pride**, **Faith**, **Willpower**, **Menace**, **Life Events**, **Holdings and Territory**, **Riches in the Land**, **Strange Devices**, and **Larger-Than-Life Play**.

The rule is simple:

- Use analytical terms only while diagnosing or comparing systems.
- Use flavorful generic terms when writing engine-agnostic YZE rules.
- Use genre-specific terms when creating a finished game.

If a term sounds like it came from a spreadsheet, a product spec, or an AI framework, rename it before it reaches a new game.

## 2. The three-name rule

Every important concept should have three possible names.

| Register | Use when | Example |
| --- | --- | --- |
| Analytical name | auditing, comparing, validating, explaining design | metacurrency, action economy, primitive, resource loop |
| Flavorful generic name | writing reusable YZE rules that can fit many genres | Willpower, action budget, pattern, work-and-haul |
| Genre-specific name | writing a finished game | Grit, Faith, Soulfire, Battery, Blood, Heat, Cred, Luck |

**Rule:** never stop at the analytical name. The analytical name is a scaffold. The flavorful generic name is the reusable rule. The genre-specific name is the game.

## 3. The forbidden blandness list

These terms may appear in internal audits and math notes, but they should not be the primary names of rules in a new game.

| Avoid as rule name | Flavorful generic replacement | Notes |
| --- | --- | --- |
| metacurrency | Willpower | Use the game's own name once chosen: Faith, Grit, Luck, Momentum, Nerve, Spirit. |
| protected dial | Pride, Faith, Inner Fire | Name the belief or courage, not the UI. |
| dial | choice, rule choice, lever, setting | "Dial" is acceptable in internal design catalogs, not in rulebook-facing naming. |
| parameter | choice, rating, limit, rank | Use the thing's name. |
| mechanic | rule, move, procedure, trick | "Mechanic" is design talk. |
| subsystem | rule set, rules module, chapter, path | Use "system" only when the book itself would say it. |
| framework | rules, method, way, path | "Framework" feels like a work presentation. |
| primitive | pattern | Use "pattern" for reusable design pieces. |
| advanced composition | pattern blend, combined pattern | Better: name the actual thing, e.g. "Heat," "Crew Stations." |
| schema | sheet, entry, form, write-up | Character sheet, holding sheet, tradition sheet. |
| interface | what it needs / what it gives | Use concrete input/output names. |
| payload | effect, result, fallout, blast, curse, sign | "Payload" only fits missiles/software. |
| cost vector | price, cost, burden, drawback | Keep it human. |
| resource loop | supply, upkeep, hunger, work, trade | Name the lived pressure. |
| extraction loop | Riches in the Land | Or survey, work, yield, hauling, depletion. |
| domain | holding, territory, realm, turf, household, command | "Domain" is too abstract unless the genre uses it. |
| asset node | holding, claim, site, outpost, road, cache | Name the place or thing. |
| population unit | family, workers, crew, settlers, households | Use people-words. |
| pressure pool | Menace | Or Heat, Doom, Wrath, Suspicion, Alarm. |
| buffer | Vigor, Resolve, Guard, Breath | Name what is being spent. |
| action economy | action budget | In rules text: "You get one SLOW action and one FAST action." |
| exploit surface | abuse check, loophole check | Internal only. |
| validation | playtest check, final check, safety check | Keep "validation" in reference files; use "check" in game text. |
| taxonomy | kinds, families, list | Use taxonomy only in internal references. |
| compatibility profile | game style, campaign style, genre style | Internal routing term only. |
| engine profile | game style | Internal routing term only. |
| candidate mechanic | possible rule | Internal ideation term only. |
| simulation-lite | quick math | Internal validation term only. |

## 4. Flavorful generic names

These are the preferred generic names for reusable YZE design. Use them before inventing a more specific game term.

| System | Flavorful generic name | What it covers |
| --- | --- | --- |
| People category | Race | fantasy peoples, species, lineage, caste, strain, model, uplift |
| Deep character past | Life Events | childhood, turns, scars, debts, mustering out, unfinished business |
| Skill purpose | What Skills Do | roll triggers, success, failure, extra success, trouble |
| Character resource | Willpower | the inner reserve spent to push, power talents, survive, cast, endure |
| Peak identity die | Pride | once-session heroic or personal conviction die |
| Belief pool | Faith | conviction as spendable strength |
| GM pressure | Menace | visible danger budget for larger-than-life play |
| Advancement | Talents and Paths | race talents, profession talents, path talents, gifts, tricks, secrets, masteries |
| Base and land | Holdings and Territory | claims, families, workers, roads, watchtowers, control |
| Resources | Riches in the Land | survey, deposits, caches, fields, yield, hauling, depletion |
| Inventions | Strange Devices | contraptions, alchemy, gadgets, prototypes, misfires |
| Powers | Powers and Traditions | disciplines, rites, tools, signs, backlash, teachers |
| Sacrifice | Blood Price | self-harm, victims, memory, years, land scars, taboo power |
| Linked power | Circle of Power | cabals, choirs, teams, synchronized crews, shared backlash |
| Heroic mode | Larger-Than-Life Play | vigor, resolve, menace, extras, mobs, champions |
| Resource dwindling | Supply Die | resource die for food, water, arrows, oxygen, fuel, ammo |

## 5. Genre-specific naming procedure

When creating a new YZE game, name the game before polishing the math.

1. **State the game's promise.** Write one sentence: "This game is about ___ risking ___ to gain ___."
2. **Name the four attributes.** Each attribute should sound like the genre's way of being strong, quick, sharp, or human.
3. **Name the four wounds.** Each attribute needs a damage name: Hurts, Shakes, Doubts, Vexes are the West model. Invent equivalents.
4. **Name Willpower.** Ask what keeps a character going when they should stop: Faith, Grit, Nerve, Breath, Luck, Signal, Rage, Hope, Battery, Blood, Reputation.
5. **Name the people-category.** Use Race if classic fantasy fits. Otherwise pick a genre word: Lineage, Model, House, Colony, Tribe, Strain, Shell, Bloodline.
6. **Name professions.** These should be roles players want to say aloud: Cutter, Sawbones, Marshal, Broker, Pilot, Warden, Saint, Rust-Eater.
7. **Name the travel day.** Quarter Day, Shift, Watch, Bell, Leg, Drift, March, Cycle, Rotation.
8. **Name the holding.** Stronghold, ranch, ship, lodge, company, family, temple, precinct, hideout, clinic, cell.
9. **Name the GM pressure.** Menace is the generic default. Genre names include Heat, Doom, Wrath, Alarm, Suspicion, Static, Infection, Debt.
10. **Read the names aloud together.** If they sound like the same game, keep them. If one sounds like a different genre, rename it.

| --- | --- | --- |
| What do players spend to push or power special acts? | Willpower | Faith, Grit, Nerve, Breath, Spark, Luck, Resolve, Soul, Battery |
| What happens when the pool is empty? | Shaken | Hollow, Spent, Rattled, Drained, Burned Out, Untethered |
| What does the GM spend to push back? | Menace | Heat, Doom, Wrath, Alarm, Static, Suspicion, Hunger |
| What is the cost face called? | Bane | Trouble, Omen, Glitch, Crack, Blood, Static, Backlash |

**Naming check:** the pool name should tell players what kind of courage the game respects.

## 8. Final naming check

Run this before finishing any new YZE game or major rules module.

- **Sheet check:** Does the character sheet contain mostly flavorful game terms, not design terms?
- **Read-aloud check:** Can a player say the core resource name with a straight face?
- **Genre check:** Could this name belong to three unrelated genres? If yes, make it more specific.
- **Portability check:** Could this generic reference name be reskinned into at least three genres? If no, it is too specific for `yze-design`.
- **Cost check:** Does every powerful name carry a price in the rules? Blood Price, Menace, Heat, Doom, and Faith should do something.
- **False-flavor check:** Is the name flavorful but mechanically empty? If yes, either give it rules or cut it.
- **No-framework check:** Search for metacurrency, protected dial, subsystem, primitive, parameter, payload, schema, interface, framework, domain, validation, and taxonomy in player-facing text. Rename or remove.
- **Specificity check:** Do not call everything "Spirit," "Edge," "Focus," or "Power." The engine gets its flavor from precise nouns.

**Final rule:** the best YZE names are plain words made dangerous by rules. Pride is not technical. Faith is not technical. Willpower is not technical. Menace is not technical. They work because the table already knows what they feel like before the dice explain what they do.
