# Rituals, Projects & Research — Long Work Under Risk

> **STATUS: WORKSHOP MODULE.** A generic long-form project engine for rituals, inventions, repairs, cures, expeditions, legal cases, and research. Built from P3 success menus, P6 activity blocks, P4 mishaps, and AC12.

## 1. Origin

- **Source patterns:** P3, P4, P6, P7.
- **Operators:** Domain Transfer + Fusion.
- **Target psychology:** progress is earned, delay is dangerous.
- **Problem solved:** important work should be more than one roll, but less than freeform GM fiat.

## 2. Generic design space

A project has a rank, threshold, cadence, required workspace, risk family, and output tags. Each work period rolls; successes bank progress; surplus buys qualities.

### Procedure

1. Define output and rank 1-6.
2. Set threshold: rank x 3 progress by default.
3. Set cadence: Shift, day, week, season.
4. Name requirements: tools, site, assistant, text, specimen, funding.
5. Roll each cadence. Success banks progress; pushed banes may trigger mishap.
6. At threshold, spend surplus on quality tags.

### Project mishap

| D6 | Mishap | Effect |
| --- | --- | --- |
| 1 | Wasted materials | Lose one requirement or pay again. |
| 2 | False lead | Next roll at −1 unless a new source is found. |
| 3 | Injury/strain | Take 1 relevant attribute damage or Stress. |
| 4 | Exposure | Heat/Attention +1. |
| 5 | Rival moves | Another actor gains progress or leverage. |
| 6 | Breakthrough at cost | Bank +1 progress and add a flaw tag. |

### Project action menu

Each work period, choose one activity. Multiple PCs may contribute, but only one lead roll banks progress unless the project is explicitly a team project.

| Activity | Roll | Success | Extra ⚔ |
| --- | --- | --- | --- |
| **RESEARCH** | LORE / science / investigation | bank 1 progress or reveal requirement | identify risk, reduce mishap severity |
| **BUILD** | CRAFT / tech / ritual skill | bank 1 progress | add quality tag at completion |
| **PROCURE** | social / economy / survival | secure missing requirement | reduce cost or avoid Debt |
| **TEST** | relevant risky skill | bank 2 progress | expose flaw before completion |
| **SECURE** | stealth / ward / guard | prevent rival/Heat tick | create safe workspace tag |
| **RESTORE** | healing / maintenance | clear one project Strain | step resource die up |

### Requirement menu

Use 2-4 requirements for ordinary projects, 5+ only for campaign-defining work.

| Requirement | What it does | Failure pressure |
| --- | --- | --- |
| **Workspace** | enables progress | work at −2 or mishap on any push |
| **Rare material** | gates completion | can progress to threshold but not finish |
| **Text / plan / formula** | reveals correct method | wrong-method mishaps on 1-2 |
| **Expert / assistant** | adds +1 or unlocks rank | Debt, Influence, or Bond Strain |
| **Time window** | creates urgency | missed window adds rank or changes output |
| **Power source** | fuels high-rank output | Corruption, Heat, or resource die step |

### Quality and flaw tags

At completion, surplus successes buy qualities. Mishaps or rushed work add flaws. Use tags, not new rule sets.

| Quality cost | Tag | Effect |
| --- | --- | --- |
| 1 ⚔ | **Durable** | resists first break/decay |
| 1 ⚔ | **Efficient** | costs one less resource per use/session |
| 1 ⚔ | **Subtle** | creates less Heat or attention |
| 2 ⚔ | **Potent** | +1 die or +1 effect in narrow use |
| 2 ⚔ | **Flexible** | works in one adjacent circumstance |
| 3 ⚔ | **Exceptional** | counts as one rank higher for one axis |

| Flaw | Effect |
| --- | --- |
| **Hungry** | consumes extra resource or attention |
| **Loud** | raises Heat/Threat on use |
| **Fragile** | breaks on pushed failure |
| **Bound** | works only for named user/place/time |
| **Unstable** | mishap on 3+ banes or severe failure |

### Resolution checks

- **Rank check:** does the output exceed its rank benchmark? Raise rank or add a flaw.
- **Time check:** does each roll represent enough fictional labor to matter?
- **Flood check:** if the project needs more than 6 rolls, convert phases into milestones.
- **Reward check:** completion should change available actions, not only grant a flat +1.

### Project archetypes

| Archetype | Primary activity | Common requirements | Mishap family | Output |
| --- | --- | --- | --- | --- |
| **Ritual** | CHANNEL / RESEARCH | site, timing, offering, text | corruption, attention, backlash | binding, curse, gate, blessing |
| **Invention** | BUILD / TEST | workshop, materials, design, expert | explosion, flaw, rival theft | device, method, tool |
| **Cure** | RESEARCH / RESTORE | patient, samples, lab, time | worsening, contamination | treatment, vaccine, antidote |
| **Expedition prep** | PROCURE / SECURE | maps, supplies, guide, permit | delay, bad intel | route, cache, advantage |
| **Legal case** | RESEARCH / CONNECT | witness, precedent, patron, proof | scandal, lost proof | injunction, pardon, conviction |
| **Construction** | BUILD / PROCURE | labor, site, materials, protection | accident, theft, delay | function, base improvement |

### Progress benchmarks

| Rank | Threshold | Scope | Example |
| --- | --- | --- | --- |
| 1 | 3 | scene/local, common materials | repair lock, prepare disguise |
| 2 | 6 | useful tool or minor cure | brew antidote, reinforce gate |
| 3 | 9 | major scenario asset | cure plague strain, build glider |
| 4 | 12 | campaign-changing asset | open warded vault, restore engine |
| 5 | 15 | legendary/rare conditions | resurrect relic, bind demon |
| 6 | 18+ | mythic, polity-scale, irreversible | stop winter, awaken city |

### Milestone structure for long projects

For Rank 4+, split work into milestones. Each milestone has a threshold and a reveal.

| Milestone | Question answered | Failure pressure |
| --- | --- | --- |
| **Theory** | can this be done? | false premise, missing lore |
| **Means** | what does it require? | rare component, patron, Debt |
| **Prototype** | what breaks first? | flaw, injury, Heat |
| **Completion** | what does it cost to finish? | sacrifice, rival, deadline |

### Project clock complications

Use alongside progress when an antagonist, deadline, or unstable force is active.

| Clock | Ticks when | Fills when |
| --- | --- | --- |
| **Rival progress** | PCs fail, delay, or make public move | rival completes or steals output |
| **Instability** | pushed rolls show banes | mishap becomes mandatory |
| **Funding dries** | PROCURE fails or Debt ignored | requirement is lost |
| **Patient worsens** | recovery phase skipped | cure must add complication |
| **Authority notices** | illegal/forbidden work | Heat threshold event |

### Worked mini-build

**Plague cure, Rank 3:** threshold 9, daily cadence. Requirements: infected sample, clean workspace, physician, old case notes. Activities: RESEARCH identifies strain, RESTORE stabilizes patients, TEST risks worsening one patient for +2 progress. Mishaps: contamination, fever spike, false cure, frightened mob, rival doctor claims credit, breakthrough at cost. Completion tags: Efficient (treats many), Stable (stores safely), Subtle (no panic), Potent (works on advanced cases).

## 3. Pressure loop

Need -> time investment -> progress/mishap -> resource drain -> output with tags.

## 4. Choices

| Choice | Fast | Slow | Psychology |
| --- | --- | --- | --- |
| Cadence | scene/Shift | week/season | montage vs campaign arc |
| Mishap | only severe pushes | any failed push | generous vs dangerous |
| Output | fixed | quality menu | procedural vs authored |

## 5. Integration points

Hooks into crafting, magic, healing, investigation, vehicles, territory, and strongholds.

## 6. Handshake

- **Prerequisites:** time units and skill rolls.
- **Inputs:** project skill, rank scale, requirements.
- **Outputs:** progress threshold, quality tags, mishaps.
- **Touched systems:** downtime, gear, powers, recovery, economy.
- **Replaces or stacks:** replaces single-roll resolution for major long work.
- **Incompatibilities:** if Spell Forging is used, use that module for powers and this for non-spell projects.

## 7. Failure modes

Too many rolls flood Willpower/Faith and table time. Too few requirements make projects feel like shopping. Requirements should create choices, not scavenger hunts.

## 8. Check notes

Use `24 §2` and `§6`. Check that progress cannot be farmed without time, risk, or cost.

## 9. Publication handoff

**Player rule:** When a task is too large for one roll, make it a project. Each work period risks time and resources, and each success moves the work toward completion.

## 10. Worked example — Cure research

Rank 3 disease cure, threshold 9, daily cadence, requires sickbed, samples, and a physician. Mishaps worsen patients, consume samples, or attract authorities.
